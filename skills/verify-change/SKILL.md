# 验证代码修改影响

**执行前先读取 `${CLAUDE_SKILL_DIR}/../_base/SKILL.md` 了解项目规范。**

## 执行流程

### 第一步：获取变更范围

1. 运行 `git diff --name-only` 获取所有已修改的文件列表
2. 运行 `git diff` 获取具体的代码变更内容
3. 如果没有未提交的变更，运行 `git diff HEAD~1 --name-only` 检查最近一次提交

### 第二步：分类分析每个变更文件

对每个变更文件，读取完整代码并按以下维度检查：

#### A. 删除分析（针对被删除的代码行）
- 提取所有被删除的方法名、变量名、组件名
- 用 Grep 在项目中搜索每个被删名称，检查是否有其他文件引用
- 特别关注：
  - `import` 语句中的引用
  - 父组件通过 `@事件名` 或 `:prop名` 的引用
  - 路由配置中的 `component` 引用
  - Vuex 中的 `mapActions`、`mapGetters` 引用
- **发现残留引用则标记为 ❌ 问题**

#### B. 修改分析（针对被修改的代码行）
- 如果修改了方法签名（参数变化），搜索所有调用处确认兼容
- 如果修改了 props 定义，搜索父组件的传值是否匹配
- 如果修改了 emit 事件名，搜索父组件的监听是否同步更新
- 如果修改了 data 字段名，检查 template 中的绑定是否同步
- **发现不一致则标记为 ❌ 问题**

#### C. 新增分析（针对新增的代码行）
- 新增的 `v-model` / `v-bind` 绑定：检查对应变量是否在 data/computed/props 中声明
- 新增的 `@click` 等事件：检查对应方法是否在 methods 中定义
- 新增的组件标签：检查是否在 components 中注册
- 新增的 import：检查模块路径是否存在（用 Glob 验证）
- 新增的表单字段：检查数据初始化（接口返回 → 赋值）和保存（提交 → 接口参数）链路
- **发现缺失则标记为 ❌ 问题**

#### D. 样式影响
- 如果修改了 CSS class 名，检查 template 中是否有引用
- 如果删除了 scoped 样式，检查是否影响子组件

### 第三步：跨文件依赖检查

- 如果修改了 API 文件（`src/apis/`），搜索所有引用该 API 的页面
- 如果修改了公共组件（`src/components/`），搜索所有使用该组件的页面
- 如果修改了路由文件，检查对应的 component 路径是否正确
- 如果修改了 store 模块，检查所有 dispatch/commit 调用

### 第四步：输出检查报告

按以下格式输出：

```
## 变更影响检查报告

### 修改文件
- file1.vue (新增 3 行, 删除 2 行)
- file2.js (修改 1 处)

### ✅ 通过项
- [描述通过的检查项]

### ❌ 发现问题
- [问题描述 + 文件:行号 + 建议修复方式]

### ⚠️ 需要关注
- [不确定是否有问题，建议人工确认的项]
```

### 第五步：修复建议

对每个 ❌ 问题，给出具体的修复代码建议，但**不自动修改**，等用户确认后再执行修复。
